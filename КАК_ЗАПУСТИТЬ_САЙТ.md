# Как выложить сайт в интернет (общий доступ)

**Бесплатно выложить можно так:**
- **Варианты 1–4** (Netlify, Vercel, Cloudflare Pages, GitHub Pages) — бесплатные тарифы, без оплаты.
- **GitHub Pages** — полностью бесплатно: репозиторий на GitHub + включить Pages в настройках. Ссылка будет вида `https://твой-логин.github.io/имя-репозитория/my-card/`.
- Платный только **свой хостинг / VPS** (раздел «Выложить сайт без посторонних сайтов») — там уже тарифы хостинга.

Самый быстрый бесплатный способ без установки программ — **Netlify** или **Cloudflare Pages** (перетащил папку → получил ссылку).

---

## Вариант 1: Netlify (рекомендуется)

1. Зайди на сайт **https://app.netlify.com**
2. Нажми **Sign up** и зарегистрируйся (можно через Google или email).
3. В личном кабинете нажми **«Add new site»** → **«Deploy manually»** (или перетащи папку в зону **«Drag and drop your site output folder here»**).
4. Открой в проводнике папку **«курсор месенджер»** (та, где лежат `my-card`, `portfolio-work-1` и т.д.) и **перетащи всю эту папку** в окно Netlify.  
   **Важно:** картинки встроены в **my-card/index.html** через скрипт (~300 КБ один файл). Перетащи папку «курсор месенджер» целиком — отдельные файлы картинок не нужны, всё грузится сразу и гарантированно.
5. Подожди 1–2 минуты. Netlify выдаст ссылку вида **https://случайное-имя-123.netlify.app**
6. Твоя карточка (сайт-визитка) будет по адресу:  
   **https://твой-адрес.netlify.app/my-card/**  
   Демо-работы откроются по ссылкам с карточки (portfolio-work-1, portfolio-work-2, portfolio-work-3).

**Свой домен (имя сайта):** в настройках сайта на Netlify можно сменить «случайное-имя» на своё, например **egor-kuchulin.netlify.app** (если оно свободно).

**Если картинки не грузятся или страница «висит» — возможно, дело в Netlify:**
- При **ручной загрузке** (Drag and drop) иногда не подхватываются вложенные папки или большие файлы, из‑за этого картинки или часть страницы не отображаются.
- **Что попробовать:** задеплоить тот же проект на **Vercel** (см. Вариант 2) — там загрузка папки часто ведёт себя стабильнее.
- Либо связать Netlify с **GitHub**: создать репозиторий, залить туда папку проекта и в Netlify выбрать «Import from Git». Тогда Netlify будет собирать сайт из репозитория, и файлы не теряются.

## Вариант 2: Vercel

1. Зайди на **https://vercel.com** и зарегистрируйся.
2. **Add New** → **Project** → **Upload** (или перетащи папку **«курсор месенджер»**).
3. Деплой создаст ссылку. Твоя страница: **https://твой-проект.vercel.app/my-card/**

*(Если Vercel недоступен — используй Вариант 3 или 4.)*

---

## Вариант 3: Cloudflare Pages (часто доступен)

1. Зайди на **https://dash.cloudflare.com**.
2. В левом меню открой **«Рабочие элементы и Страницы»** (Workers & Pages). На главной дашборда можно нажать карточку **«Рабочие элементы и Страницы»** → **«Начните создавать»**.
3. Нажми **Create** → **Pages** → **Upload assets** (загрузка файлов).
4. В поле **Project name** введи имя, например **portfolio**.
5. Перетащи в зону загрузки **всю папку «курсор месенджер»** (та, где лежат `my-card`, `portfolio-work-1`, `index.html` и т.д.).
6. Дождись окончания деплоя. Ссылка будет вида **https://portfolio-xxx.pages.dev**
7. Карточка откроется по адресу: **https://portfolio-xxx.pages.dev/my-card/**

**Если Cloudflare показывает предупреждение про TypeScript или build:** для этого проекта его можно игнорировать — сайт статичный, сборка на сервере не нужна. Просто нажми **«Развертывание»** (Deploy).

**Если кнопка «Развёртывание» не срабатывает:**
- Убедись, что в поле **Project name** / **Имя проекта** введено имя (латиницей, без пробелов), например **portfolio**.
- Попробуй другой браузер (Chrome, Edge, Firefox) или режим инкогнито — иногда мешают расширения.
- Если не помогает — задеплой через **GitHub**: нажми **«Подключите GitHub»**, создай репозиторий, залей туда папку проекта, затем в Cloudflare выбери этот репозиторий для Pages. Кнопка развёртывания при загрузке через Git не нужна — деплой запускается автоматически после push.

## Вариант 4: GitHub Pages (если установлен Git)

1. Установи Git с https://git-scm.com/download/win
2. Создай репозиторий на GitHub, залей туда папку проекта.
3. В настройках репозитория: **Settings** → **Pages** → Source: **Deploy from a branch** → ветка **main** → папка **/ (root)** → Save.
4. Через пару минут сайт будет по адресу **https://твой-логин.github.io/имя-репозитория/**  
   Карточка: **https://твой-логин.github.io/имя-репозитория/my-card/**

**Если через браузер не все файлы добавляются** — залей проект через Git с компьютера:
1. Установи Git (если ещё не установлен): https://git-scm.com/download/win
2. Открой **PowerShell** или **Командную строку**. Перейди на рабочий стол (или туда, где удобно):
   ```bash
   cd $HOME\Desktop
   ```
3. Клонируй свой репозиторий (подставь свой логин и имя репо):
   ```bash
   git clone https://github.com/Egrassvy/my-card23.git
   cd my-card23
   ```
4. **Важно:** скопируй **все файлы и папки** из папки «курсор месенджер» в папку `my-card23` (проводник: открыть обе папки → в «курсор месенджер» Ctrl+A → Ctrl+C → открыть `my-card23` → Ctrl+V). Внутри `my-card23` должны появиться `index.html`, папки `my-card`, `portfolio-work-1`, `portfolio-work-2`, `portfolio-work-3` и остальное. Без этого шага на GitHub уйдёт пустой репозиторий.
5. В PowerShell (в папке `my-card23`) выполни:
   ```bash
   git add .
   git commit -m "Добавлены все файлы сайта"
   git push
   ```
6. При первом `git push` GitHub попросит логин и пароль. Пароль — это **токен**: зайди на GitHub → **Settings** (твоего профиля) → **Developer settings** → **Personal access tokens** → создай токен с правом **repo**, скопируй его и вставь вместо пароля.
7. После успешной отправки включи **Settings** → **Pages** в репозитории (см. выше).

## Выложить сайт без Netlify/Cloudflare/Vercel (свой хостинг)

Если не хочешь зависеть от зарубежных сервисов, можно разместить сайт на **обычном хостинге** или **своём сервере**.

**Вариант А: Российский хостинг (статика или «сайт-визитка»)**  
Провайдеры вроде **Beget**, **Timeweb**, **Reg.ru**, **Sprinthost** дают недорогой хостинг с панелью и FTP.

1. Оформи тариф с доступом по FTP (или «Хостинг сайтов»).
2. В панели хостинга найди **файловый менеджер** или подключись по **FTP** (FileZilla и т.п.).
3. В корень сайта (обычно папка `public_html` или `www`) залей **содержимое** папки «курсор месенджер»:  
   — файл `index.html` (из корня проекта) — в корень;  
   — папки `my-card`, `portfolio-work-1`, `portfolio-work-2`, `portfolio-work-3` — туда же.
4. Если у тебя есть домен, привяжи его к хостингу в панели. Сайт будет открываться по твоему домену, например **https://твой-сайт.ru/my-card/**.

Плюсы: свой домен, нет зависимости от Netlify/Cloudflare, часто стабильнее открывается в РФ.

**Вариант Б: Свой сервер (VPS)**  
Если арендуешь VPS (например Timeweb, Selectel, DigitalOcean), можно поднять на нём веб-сервер (nginx/apache), залить туда папку с сайтом и отдавать её по своему домену. Требует настройки сервера; подходит, если хочешь полный контроль.

## Если ничего из этого недоступно: показ локально

Можно открыть карточку у себя на компьютере и показывать скриншот или экран:
1. Открой папку **my-card** в проводнике.
2. Дважды щёлкни по **index.html** — откроется в браузере по адресу `file:///...`
3. Либо в VS Code / Cursor: правый клик по **index.html** → **Open with Live Server** (если установлено расширение) — тогда откроется как локальный сайт с адресом типа `http://127.0.0.1:5500/my-card/`.

## Что получится после деплоя

- По корневой ссылке (например **https://xxx.netlify.app/**) откроется переход на карточку.
- **Твоя карточка с услугами и работами:** **https://xxx.netlify.app/my-card/**
- Ссылки «Открыть демо» на карточке будут вести на твои работы (кофейня, фотостудия, многостраничный сайт).

Эту ссылку на **my-card** можно указывать в профиле FL.ru, в Telegram и в резюме.
